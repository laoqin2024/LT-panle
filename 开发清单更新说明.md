# 开发清单更新说明

## 📋 如何使用开发清单

### 1. 更新频率

- **每日更新**: 完成功能后立即更新
- **每周总结**: 每周五更新周进度
- **里程碑更新**: 完成重要功能模块后更新

### 2. 更新方式

#### 标记完成
```markdown
- [x] 功能名称  # 已完成
```

#### 标记进行中
```markdown
- [ ] 功能名称  # 进行中（在描述中说明）
```

#### 标记待开发
```markdown
- [ ] 功能名称  # 待开发
```

### 3. 更新内容

#### 必须更新的部分
1. **最后更新日期** - 每次更新都要修改
2. **完成的功能** - 添加新完成的功能
3. **进行中的功能** - 更新进度
4. **更新日志** - 记录本次更新的内容

#### 可选更新的部分
1. **统计数据** - 定期更新数字
2. **开发计划** - 根据实际情况调整
3. **技术债务** - 发现新问题时添加

### 4. 更新示例

#### 完成一个功能后
```markdown
### 2026-01-11
- ✅ 完成业务站点列表API（GET /api/sites）
- ✅ 完成业务站点创建API（POST /api/sites）
- 🟡 开始业务站点更新API开发
```

#### 修复一个BUG后
```markdown
### 2026-01-11
- 🐛 修复业务站点列表分页问题
- 🐛 修复前端站点状态显示错误
```

### 5. 优先级标记

在功能描述前添加优先级标记：

- 🔴 **高优先级** - 核心功能，必须完成
- 🟡 **中优先级** - 重要功能，建议完成
- 🟢 **低优先级** - 可选功能，有时间再完成

### 6. 状态图标

- ✅ **已完成** - 功能已实现并通过测试
- 🟡 **进行中** - 正在开发
- 🔴 **待开发** - 尚未开始
- ⚠️ **有问题** - 遇到问题需要解决
- 🐛 **BUG** - 发现的问题

### 7. 版本标记

重要功能完成后，可以添加版本标记：

```markdown
### v0.1.0 (2026-01-10)
- ✅ 完成用户认证系统
- ✅ 完成数据库初始化
```

---

## 📝 更新检查清单

每次更新前检查：

- [ ] 更新日期是否正确
- [ ] 完成的功能是否已标记
- [ ] 统计数据是否准确
- [ ] 更新日志是否记录
- [ ] 优先级是否正确
- [ ] 状态图标是否正确

---

## 🎯 建议

1. **及时更新**: 完成功能后立即更新，避免遗忘
2. **详细记录**: 记录关键信息，便于后续查看
3. **定期回顾**: 每周回顾一次，调整开发计划
4. **团队协作**: 如果是团队项目，确保所有成员都更新

---

**提示**: 保持开发清单的准确性和及时性，有助于项目管理和进度跟踪。
